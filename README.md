# MNIST Number Predictor

A handwritten digit recognition app built with a PyTorch DNN trained on MNIST, exported to ONNX, and deployed as a browser-based drawing demo, no server required.

## Demo

Run in a browser through localHost, draw a digit (0–9) on the canvas, and the model predicts it in real time.

## Project Structure

```
mnist_app/
├── Source code.ipynb     # Model training, testing, and ONNX export
├── index.html            # Browser demo using ONNX Runtime Web
└── my_mnist_model.onnx   # Exported model (generated by the notebook)
```

## Model Architecture

A fully-connected DNN (no convolutions) built in PyTorch:

| Layer | Size |
|-------|------|
| Input | 784 (28×28 flattened) |
| Hidden A | 235 |
| Hidden B | 72 |
| Hidden C | 22 |
| Output | 10 (digits 0–9) |

Activations: Sigmoid on hidden layers, Softmax on output.
Optimizer: Adam (lr=0.0005) | Loss: CrossEntropyLoss | Epochs: 10 | Batch size: 30
**Test accuracy: 96.37%**

## How to Run the Notebook

### Prerequisites

```bash
pip install torch torchvision numpy matplotlib scikit-learn onnx
```

### Steps

1. Run all cells in `Source code.ipynb` in order:
   - Cells 1–3: Load MNIST data and create dataloaders
   - Cell 5: Visualize sample digits
   - Cells 7–10: Define and initialize the model
   - Cell 12: Train for 10 epochs
   - Cell 14: Evaluate on the test set
   - Cell 16: Save weights to `my_mnist_model.pth`
   - Cells 17–18: Install `onnx` and export to `my_mnist_model.onnx`

2. The exported `my_mnist_model.onnx` is what `index.html` loads.

## How to Run the Demo

> **Note:** Most browsers block local file loading due to CORS restrictions. If the model fails to load, serve the files with a local HTTP server:
> ```bash
> python -m http.server 8000
> ```
> Then open `http://localhost:8000` in your browser.

## How the Demo Works

- The canvas is 28×28 pixels internally, scaled up 10× visually (280×280) for drawing.
- When you lift the mouse/finger, the drawn digit is:
  1. Cropped to its bounding box
  2. Scaled to fit a 20×20 box (matching MNIST's formatting)
  3. Centered in the 28×28 canvas
  4. Flattened to a 784-element float array and normalized to [0, 1]
- The preprocessed input is fed to the ONNX model via [ONNX Runtime Web](https://onnxruntime.ai/docs/get-started/with-javascript/web.html).
- The output is 10 softmax probabilities, displayed as a bar chart.
